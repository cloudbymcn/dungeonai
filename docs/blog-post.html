DungeonAI

Como construir um RPG text adventure com IA generativa na AWS usando Amazon Bedrock, Strands Agents, DynamoDB, S3 e Streamlit.

Amazon Bedrock Claude 3.5 Haiku Titan Image Generator Strands Agents SDK DynamoDB S3 Streamlit Python

Neste post, vou mostrar como construi o DungeonAI — um RPG text adventure onde a IA narra a historia, gera as cenas em imagem, e gerencia o estado do jogo (HP, inventario, XP) tudo rodando na AWS.

O jogador escolhe acoes ou digita livremente, e o agente de IA responde com narrativa imersiva + uma imagem gerada para cada cena. O estado do jogo persiste no DynamoDB, e as imagens ficam no S3.

Repositorio: Todo o codigo esta disponivel no GitHub → cloudbymcn/dungeonai

Arquitetura

Jogador → Streamlit → Strands Agent → Amazon Bedrock | +--------------+--------------+ | | | Claude 3.5 Haiku Titan Image V2 DynamoDB (narrativa) (cenas) (game state) | S3 Bucket (imagens salvas)

O fluxo de cada turno:

  1. Jogador escolhe ou digita uma acao no Streamlit
  2. O Strands Agent carrega o game state do DynamoDB
  3. Claude 3.5 Haiku gera a narrativa + prompt de imagem + mudancas de estado
  4. Titan Image Generator V2 (Nova Canvas) gera a imagem da cena
  5. Imagem e salva no S3, estado atualizado no DynamoDB
  6. Tudo retorna ao frontend: narrativa + imagem + novo estado + sugestoes

Servicos AWS utilizados

Servico Funcao Custo por sessao (~15 turnos)
Amazon Bedrock — Claude 3.5 Haiku Gera narrativa, estado, sugestoes de acao ~$0.02
Amazon Bedrock — Titan Image Generator V2 Gera imagem da cena a cada turno ~$0.40–0.80
DynamoDB Persiste game state (HP, inventario, localizacao) < $0.01
S3 Armazena imagens geradas < $0.01
Dica: Durante o desenvolvimento, comente a chamada do generate_scene no dungeon_master.py e teste so a narrativa primeiro. A geracao de imagem e o custo mais alto.

Passo a passo

1

Configurar os models no Amazon Bedrock

Os models que usamos ja estao disponiveis no Model Catalog do Bedrock — nao e necessario solicitar acesso manualmente.

  1. Acesse o console AWS → procure Amazon Bedrock
  2. No menu lateral, clique em Model catalog
  3. Confirme que os seguintes models estao disponiveis na regiao us-east-1:
    • Anthropic → Claude 3.5 Haiku
    • Amazon → Titan Image Generator V2
Todos os models listados no Model Catalog ja estao liberados. Basta confirmar que aparecem na sua regiao.
2

Criar a tabela no DynamoDB

A tabela armazena o game state (HP, inventario, localizacao, historico) de cada sessao.

  1. Console AWS → DynamoDB → Create table
  2. Table name: dungeonai-game-sessions
  3. Partition key: session_id (tipo String)
  4. Sort key: deixe em branco
  5. Table settings → Customize settings
  6. Capacity mode → On-demand (paga so pelo uso)
  7. Clique Create table
Verificacao: Clique na tabela → aba Explore table items → deve estar vazia.
3

Criar o bucket S3

O bucket armazena as imagens geradas pelo Nova Canvas a cada turno do jogo.

  1. Console AWS → S3 → Create bucket
  2. Bucket name: dungeonai-scenes
    Se o nome ja estiver em uso (S3 e global), use dungeonai-scenes-SEU_ACCOUNT_ID e atualize a variavel S3_BUCKET em generate_scene.py
  3. Region: us-east-1
  4. Block Public Access: tudo marcado (as imagens sao acessadas pelo backend via SDK)
  5. Clique Create bucket

Criar o System Prompt do Dungeon Master

O prompt define a personalidade da IA, as regras de combate, o mundo do jogo, e o formato de output. Esse e o coracao do agente — quanto mais detalhado, melhor a experiencia.

📜 system_prompt.txt agent/prompts/system_prompt.txt 
You are Eldrath, an ancient and theatrical Dungeon Master narrating a dark fantasy RPG.
Your voice is dramatic yet concise — like a bard recounting legends by firelight.

============================
NARRATIVE RULES
============================
- Write in Brazilian Portuguese, second person ("Voce entra na caverna...")
- Each narrative beat: 2-3 short paragraphs (max 600 characters total)
- First paragraph: immediate consequence of the player's action
- Second paragraph: what the player now perceives (senses, atmosphere, danger)
- Optional third paragraph: only for dramatic moments (combat result, discovery, death)
- Use vivid sensory details: sounds echoing, cold air, the smell of damp stone, flickering light
- Vary sentence length — mix short punchy lines with longer atmospheric ones
- Never repeat the player's action back verbatim; show the result, not the intent
- End every beat with exactly 3 suggested actions (short, 5-10 words each)
- Suggested actions should be meaningfully different: one cautious, one bold, one creative

============================
TONE & PERSONALITY
============================
- You are not neutral — you have personality. React to player choices:
  - Reckless actions → ominous foreshadowing ("As sombras parecem se mover...")
  - Clever actions → subtle praise woven into narration ("Seus olhos treinados percebem...")
  - Cowardly retreats → eerie tension ("O silencio atras de voce e quase pior...")
- Use dramatic irony: hint at dangers the player hasn't discovered yet
- Maintain continuity: reference previous events and choices when relevant
- The world reacts to the player — NPCs remember, environments change

============================
COMBAT SYSTEM
============================
Players have base stats: STR 14, DEX 12, INT 10 (can improve with items/levels).

When combat occurs:
1. Roll logic: d20 + relevant stat modifier vs difficulty (easy=8, medium=12, hard=16, deadly=20)
   - STR: melee attacks, breaking things, grappling
   - DEX: ranged attacks, dodging, stealth
   - INT: magic, puzzles, identifying creatures
2. Damage: success = 5-15 dmg to enemy; failure = player takes 5-20 HP loss
3. Critical (nat 20 equivalent): double effect + bonus loot/XP
4. Critical fail (nat 1 equivalent): something goes wrong — weapon breaks, trap triggers
5. Never show dice numbers in narrative — describe outcomes dramatically
6. Award 15-30 XP for combat encounters, 10-20 XP for clever non-combat solutions

============================
PROGRESSION & ECONOMY
============================
- HP: max 100, potions restore 20-40 HP, resting restores 10 HP
- Gold: found in chests (5-20), dropped by enemies (3-10), reward for quests (20-50)
- XP: level up every 100 XP. Each level: +10 max HP
- Items: weapons have tiers (iron → steel → enchanted), give narrative combat bonuses
- Inventory limit: 8 items. If full, player must drop something
- Death is permanent for the session — make it dramatic and memorable

============================
GAME WORLD: THE CAVERNS OF ELDRATH
============================
An ancient underground kingdom where crystals pulse with residual magic from a fallen civilization.

KEY LOCATIONS (reveal progressively):
- Entrada das Cavernas: mossy stone arch, cold wind, faint purple glow from within
- Salao dos Cristais: massive cavern with crystalline pillars that hum with energy
- Ponte do Abismo: narrow stone bridge over a bottomless chasm, wind howling from below
- Forja Abandonada: dwarven smithy with still-warm coals and scattered tools
- Santuario Submerso: flooded temple with bioluminescent water and submerged statues
- Trono de Eldrath: final chamber, throne of black crystal, the source of all magic

CREATURES:
- Sombras Rastejantes: shadow creatures, weak but attack in groups (easy)
- Golem de Cristal: slow but devastating, vulnerable to magic (hard)
- Aranhas Cavernosas: ambush predators, webs slow movement (medium)
- O Guardiao: ancient construct protecting the throne room (deadly)
- Espiritos dos Antigos: ethereal beings, can be hostile or helpful depending on approach

NPCS:
- Lira: a trapped explorer, can become an ally if rescued (found in Salao dos Cristais)
- O Mercador Cego: mysterious trader who appears randomly, sells potions and info for gold

============================
IMAGE PROMPT RULES
============================
When generating the image_prompt field, write ONE English sentence (max 30 words) describing
the visual scene for an AI image generator. Rules:
- Style: dark fantasy digital painting, atmospheric lighting, rich colors
- Focus on: environment first, then key objects, then mood/lighting
- NEVER include: text, UI elements, people/characters, hands, faces
- NEVER use generic prompts — each scene should feel unique

============================
OUTPUT FORMAT
============================
You MUST respond with valid JSON only. No markdown, no code fences, no explanation outside the JSON.

{
    "narrative": "Narrative text in Brazilian Portuguese...",
    "image_prompt": "One English sentence, max 30 words, dark fantasy style",
    "state_changes": {
        "hp_delta": 0,
        "gold_delta": 0,
        "xp_delta": 0,
        "new_items": [],
        "removed_items": [],
        "new_location": "current or new location name"
    },
    "suggested_actions": ["acao curta 1", "acao curta 2", "acao curta 3"]
}

Pontos-chave do prompt:

  • Personalidade: O DM (Eldrath) reage ao estilo do jogador — ousadia, covardia, inteligencia
  • Combate d20: Sistema de dados simulado com STR/DEX/INT, sem mostrar numeros na narrativa
  • Mundo detalhado: 6 locais, 5 criaturas, 2 NPCs — tudo progressivo
  • Output estruturado: JSON com narrativa, prompt de imagem, mudancas de estado e sugestoes

Criar as Tools do Agente

O agente usa 3 tools: uma para narrar a historia (Claude Haiku), uma para gerar a imagem da cena (Titan/Nova Canvas), e uma para gerenciar o estado do jogo (DynamoDB).

📖 narrate_story.py — Narrativa via Claude Haiku agent/tools/narrate_story.py 
import json
import boto3
from strands import tool

bedrock = boto3.client("bedrock-runtime", region_name="us-east-1")

MODEL_ID = "us.anthropic.claude-3-5-haiku-20241022-v1:0"


@tool
def narrate_story(
    player_action: str,
    game_state: dict,
    scene_history: list[str] = None,
) -> dict:
    """Generate the next narrative beat based on the player's action
    and current game state.

    Args:
        player_action: What the player wants to do
        game_state: Current game state with hp, gold, xp, level, inventory, location
        scene_history: List of the last 3 scene summaries for continuity

    Returns:
        dict with narrative text, image prompt, state changes, and suggested actions
    """
    history_context = ""
    if scene_history:
        history_context = "Cenas anteriores:\n" + "\n".join(
            f"- {s}" for s in scene_history[-3:]
        )

    prompt = f"""Voce e o Dungeon Master. Baseado na acao do jogador e no estado atual,
narre o proximo acontecimento.

Estado atual do jogo:
- HP: {game_state.get('hp', 100)}/100
- Gold: {game_state.get('gold', 10)}
- XP: {game_state.get('xp', 0)}
- Level: {game_state.get('level', 1)}
- Inventario: {json.dumps(game_state.get('inventory', []), ensure_ascii=False)}
- Localizacao: {game_state.get('location', 'Entrada das Cavernas de Eldrath')}

{history_context}

Acao do jogador: {player_action}

Responda APENAS com JSON valido (sem markdown, sem ```):
{{
    "narrative": "Narrativa em portugues, 2-4 paragrafos, segunda pessoa",
    "image_prompt": "One English sentence describing the visual scene, fantasy art style",
    "state_changes": {{
        "hp_delta": 0,
        "gold_delta": 0,
        "xp_delta": 0,
        "new_items": [],
        "removed_items": [],
        "new_location": "nome do local atual ou novo"
    }},
    "suggested_actions": ["acao 1", "acao 2", "acao 3"]
}}"""

    response = bedrock.converse(
        modelId=MODEL_ID,
        messages=[{"role": "user", "content": [{"text": prompt}]}],
        inferenceConfig={"maxTokens": 1024, "temperature": 0.8},
    )

    raw = response["output"]["message"]["content"][0]["text"]

    try:
        result = json.loads(raw)
    except json.JSONDecodeError:
        result = {
            "narrative": raw,
            "image_prompt": "A mysterious dark fantasy cave with glowing crystals, atmospheric lighting",
            "state_changes": {
                "hp_delta": 0, "gold_delta": 0, "xp_delta": 0,
                "new_items": [], "removed_items": [],
                "new_location": game_state.get("location", "Desconhecido"),
            },
            "suggested_actions": [
                "Explorar os arredores",
                "Voltar pelo caminho anterior",
                "Descansar por um momento",
            ],
        }

    return result
🎨 generate_scene.py — Imagem via Titan/Nova Canvas agent/tools/generate_scene.py 
import json
import base64
import uuid
import boto3
from strands import tool

bedrock = boto3.client("bedrock-runtime", region_name="us-east-1")
s3 = boto3.client("s3", region_name="us-east-1")

IMAGE_MODEL_ID = "amazon.titan-image-generator-v2:0"
S3_BUCKET = "dungeonai-scenes"


@tool
def generate_scene(image_prompt: str, session_id: str) -> dict:
    """Generate a scene image based on the narrative description.

    Args:
        image_prompt: English description of the scene to generate
        session_id: Current game session ID for organizing images in S3

    Returns:
        dict with s3_key and image_bytes (base64) of the generated scene
    """
    styled_prompt = (
        f"{image_prompt}, "
        "dark fantasy art style, "
        "detailed environment, "
        "dramatic lighting, "
        "painterly illustration, "
        "no text, no UI elements, no people"
    )

    body = json.dumps(
        {
            "taskType": "TEXT_IMAGE",
            "textToImageParams": {"text": styled_prompt},
            "imageGenerationConfig": {
                "numberOfImages": 1,
                "quality": "standard",
                "cfgScale": 7.0,
                "height": 768,
                "width": 1280,
                "seed": 0,
            },
        }
    )

    response = bedrock.invoke_model(
        modelId=IMAGE_MODEL_ID,
        contentType="application/json",
        accept="application/json",
        body=body,
    )

    result = json.loads(response["body"].read())
    image_b64 = result["images"][0]

    # Save to S3
    image_key = f"sessions/{session_id}/{uuid.uuid4().hex[:8]}.png"
    s3.put_object(
        Bucket=S3_BUCKET,
        Key=image_key,
        Body=base64.b64decode(image_b64),
        ContentType="image/png",
    )

    return {
        "s3_key": image_key,
        "image_base64": image_b64,
    }
💾 update_state.py — Game State no DynamoDB agent/tools/update_state.py 
import json
from datetime import datetime, timezone
import boto3
from strands import tool

dynamodb = boto3.resource("dynamodb", region_name="us-east-1")
table = dynamodb.Table("dungeonai-game-sessions")


def _default_state() -> dict:
    """Return a fresh game state for new sessions."""
    return {
        "hp": 100,
        "gold": 10,
        "xp": 0,
        "level": 1,
        "inventory": ["Espada de ferro", "Tocha"],
        "location": "Entrada das Cavernas de Eldrath",
        "turn": 0,
        "scene_history": [],
        "status": "active",
    }


@tool
def update_game_state(session_id: str, state_changes: dict = None) -> dict:
    """Load, update, and persist the game state in DynamoDB.

    If no state_changes are provided, returns the current state (or creates a new one).

    Args:
        session_id: Unique game session identifier
        state_changes: Changes to apply (hp_delta, gold_delta, xp_delta,
                       new_items, removed_items, new_location)

    Returns:
        The updated game state dict
    """
    # Load current state
    try:
        response = table.get_item(Key={"session_id": session_id})
        state = response.get("Item", {}).get("game_state")
        if state:
            state = json.loads(state) if isinstance(state, str) else state
        else:
            state = _default_state()
    except Exception:
        state = _default_state()

    # Apply changes
    if state_changes:
        state["hp"] = max(0, min(100, state["hp"] + state_changes.get("hp_delta", 0)))
        state["gold"] = max(0, state["gold"] + state_changes.get("gold_delta", 0))
        state["xp"] += state_changes.get("xp_delta", 0)

        new_level = 1 + state["xp"] // 100
        if new_level > state["level"]:
            state["level"] = new_level

        for item in state_changes.get("new_items", []):
            if item and item not in state["inventory"]:
                state["inventory"].append(item)

        for item in state_changes.get("removed_items", []):
            if item in state["inventory"]:
                state["inventory"].remove(item)

        if state_changes.get("new_location"):
            state["location"] = state_changes["new_location"]

        state["turn"] += 1

        if state["hp"] <= 0:
            state["status"] = "dead"

    # Persist
    table.put_item(
        Item={
            "session_id": session_id,
            "game_state": json.dumps(state, ensure_ascii=False),
            "updated_at": datetime.now(timezone.utc).isoformat(),
        }
    )

    return state

Criar o Orquestrador (Dungeon Master Agent)

O dungeon_master.py conecta tudo: cria o agente Strands com as tools e orquestra o fluxo de cada turno.

🧠 dungeon_master.py agent/dungeon_master.py 
import uuid
from pathlib import Path
from strands import Agent
from strands.models.bedrock import BedrockModel

from agent.tools.narrate_story import narrate_story
from agent.tools.generate_scene import generate_scene
from agent.tools.update_state import update_game_state

SYSTEM_PROMPT = (Path(__file__).parent / "prompts" / "system_prompt.txt").read_text()


def create_agent() -> Agent:
    """Create and return the Dungeon Master agent."""
    model = BedrockModel(
        model_id="us.anthropic.claude-3-5-haiku-20241022-v1:0",
        region_name="us-east-1",
    )
    return Agent(
        model=model,
        system_prompt=SYSTEM_PROMPT,
        tools=[narrate_story, generate_scene, update_game_state],
    )


def play_turn(agent: Agent, session_id: str, player_action: str) -> dict:
    """Execute a single game turn.

    Flow:
    1. Load current game state from DynamoDB
    2. Generate narrative based on player action (Claude Haiku)
    3. Generate scene image from the narrative (Titan Image Gen)
    4. Update and persist game state
    5. Return everything to the frontend
    """
    # Load state
    game_state = update_game_state(session_id=session_id)

    # Generate narrative
    story = narrate_story(
        player_action=player_action,
        game_state=game_state,
        scene_history=game_state.get("scene_history", []),
    )

    # Generate scene image
    scene = generate_scene(
        image_prompt=story["image_prompt"],
        session_id=session_id,
    )

    # Update state
    updated_state = update_game_state(
        session_id=session_id,
        state_changes=story["state_changes"],
    )

    # Track scene history (last 5)
    scene_summary = story["narrative"][:120]
    history = updated_state.get("scene_history", [])
    history.append(scene_summary)
    if len(history) > 5:
        history = history[-5:]
    updated_state["scene_history"] = history

    update_game_state(
        session_id=session_id,
        state_changes={"hp_delta": 0, "gold_delta": 0, "xp_delta": 0},
    )

    return {
        "narrative": story["narrative"],
        "image_base64": scene["image_base64"],
        "image_s3_key": scene["s3_key"],
        "game_state": updated_state,
        "suggested_actions": story["suggested_actions"],
    }


def new_session() -> str:
    """Start a new game session and return the session ID."""
    session_id = uuid.uuid4().hex[:12]
    update_game_state(session_id=session_id)
    return session_id

Como funciona por dentro

O ciclo de um turno

Quando o jogador clica em uma acao ou digita algo:

  1. A acao e enfileirada no session_state e o Streamlit faz um rerun
  2. No topo do rerun, detectamos a acao pendente e mostramos a tela de loading
  3. play_turn() e chamado, que:
    • Carrega o estado do DynamoDB
    • Envia o estado + acao para o Claude 3.5 Haiku via bedrock.converse()
    • Claude retorna JSON com narrativa, prompt de imagem, e mudancas de estado
    • O prompt de imagem vai para o Titan Image Generator V2 via bedrock.invoke_model()
    • A imagem e salva no S3 e retornada em base64
    • O estado e atualizado no DynamoDB
  4. Outro rerun renderiza a nova cena

Por que Strands Agents?

O Strands Agents SDK e um framework open-source da AWS para construir agentes de IA. Ele permite:

  • Definir tools como funcoes Python decoradas com @tool
  • Conectar diretamente ao Amazon Bedrock sem configuracao extra
  • Orquestrar chamadas entre LLM, ferramentas e estado

Por que Claude Haiku para narrativa?

  • Rapido: Latencia baixa para manter o jogo fluido
  • Barato: ~$0.001 por turno de narrativa
  • Bom em PT-BR: Gera narrativa natural em portugues
  • JSON confiavel: Segue o formato de output com consistencia

Troubleshooting

Erro Causa provavel Solucao
AccessDeniedException Model nao disponivel ou sem permissao IAM Verifique Model Catalog + IAM policy
ResourceNotFoundException Tabela DynamoDB nao existe Confirme o nome dungeonai-game-sessions em us-east-1
NoSuchBucket Bucket S3 nao existe Confirme o bucket e atualize S3_BUCKET se necessario
Imagem nao aparece Titan Image Gen com erro Verifique logs no terminal; teste so narrativa primeiro
streamlit: command not found Streamlit nao esta no PATH Use python -m streamlit run frontend/app.py
